数据结构的形式定义

数据结构是一个二元组
Data_Structure=(D,S)
其中：D是数据元素的有限集
S是D上关系的有限集

一对有序关系<A,B> 表示为：A→B
一对无序关系(A,B) 表示为：A-B

数据的逻辑结构

  1. 线性结构（1:1） 有且只有一个开始结点，有且只有一个终端结点，其他结点有且只有一个前驱，有且只有一个后继。
  2. 树形结构（1:n） 有且只有一个开始结点，可能有多个终端结点，其他结点只有一个前驱，可能有多个后继。
  3. 图状结构（m:n） 可能有多个开始结点，可能有多个终端结点，其他结点可能有多个前驱，可能有多个后继。
  4. 集合结构（松散）

线性结构又可分为：

  • 线性表
  • 栈
  • 队列
  • 串及数组

非线性结构即包括：树形结构，图状结构，集合结构。

数据的存储结构

数据的存储结构包括数据元素的映象方法和数据关系的映象方法。

数据元素的映象方法：
用二进制(bit)的位串表示数据元素
(456)10 = (111001000)2

数据关系的映象方法：（表示<x,y>的方法）

  1. 顺序映象：以相对的存储位置来表示数据元素之间的逻辑关系。
    逻辑关系通过物理位置来反映。
  2. 链式（非线性）映象：以附加信息（指针）表示后继关系。
    需要用一个和x附加在一起的指针来指示y的存储位置。
  3. 索引映象：不常用。
  4. 散列映象：不常用。

在高级程序设计语言中，数据的存储结构是通过数据类型来描述的。
对变量或常量说明其所属数据类型的作用是：可明确其存储结构、取值范围和可进行的操作三方面的内容。

抽象数据类型

抽象数据类型（简称ADT）是指一数据值的集合和定义在这个集合上的一组操作。
Abstract Data Type = {Objects}∪{Operations}
例如：int = {0,±1,±2,…,INT_MAX,INT_MIN}∪{+,-,×,÷,%,…}

ADT的两个特性：数据抽象和数据封装

抽象数据类型的三元组形式描述为：
ADT = (D,S,P)
其中：D是数据对象
S是D上的关系集
P是D的基本操作集

算法的基本概念

算法就是求解问题的步骤。算法具有以下五个特性：

  1. 输入：一个算法有零个或多个输入。
  2. 输出：一个算法有一个或多个输出。
  3. 确定性：基本操作都是语义明确的，不具二义性。
  4. 可行性：每一基本操作都可实现。
  5. 有穷性：基本操作的次数是有穷的，可以在有穷的时间内得到输出。

从Hailstone序列可以看出程序≠算法
一个程序不一定是算法，因为程序不一定满足有穷性。
程序中的指令必须是机器可执行的，而算法中的指令则无此限制，所以当一个算法用计算机语言来描写时，算法就是一个程序。

好算法——设计原则：

  • 正确性：能够正确处理任意规模（特殊的、一般的）的合法输入。
  • 健壮性：能辨别不合法的输入并做适当处理，而不至于非正常退出。即要具有容错处理的能力。
  • 可读性：结构清晰+准确命名+注释+…
  • 高效性：速度尽可能快；存储空间尽可能少。（既要马儿快快跑，又要马儿少吃草）

算法的效率分析

影响算法执行时间的主要因素：

  1. 算法选用的策略
  2. 问题的规模
  3. 编写程序的语言
  4. 编译程序产生的机器代码的质量
  5. 计算机执行指令的速度

一个特定算法的运行时间，它是问题规模（通常用整数量n表示）的函数。
算法时间的量度记为：
T(n) = O(f(n)) ———渐近时间复杂度
上式的数学含义：存在两个常量C和N，当n>=N时，有T(n)<=Cf(n)

大O表示法的计算规律：

  • 多项式————取最高次项去系数
    $T(n)=a_kn^k+a_{k-1}k^{k-1}+…+a_1n+a_0=O(n^k) (a_k>0)$
  • 算数级数————与末项平方同阶
    $T(n)=1+2+…+n=n(n+1)/2=O(n^2)$
  • 指数级数————比幂次高出一阶
    $T(n)=1^m+2^m+…+n^m=O(n^{m+1})$
  • 几何级数————与末项同阶
    $T(n)=a^0+a^1+…+a^n=O(a^n)$

算法的渐进时间复杂度

$T(n) = \sum_1^n$原操作(i)的语句频度
语句频度就是语句重复执行的次数

两个n×n阶矩阵相乘：

$T(n) = n^3+n^2 = O(n^3)$

T(n) = 关键操作的语句频度
关键操作往往是算法中控制结构内最深层的原操作

求下列程序段的时间复杂度：

设关键操作语句的执行次数为T(n),
则有$2^{T(n)}<=n$,
所以$T(n)<=\log_2n$
所以$T(n) = O(\log_2n) = O(logn)$ (对于对数函数，常底数无所谓)
关键操作的语句频度为：$\lfloor \log_2n \rfloor$ (向下取整)

求下列程序段的时间复杂度：

设关键操作语句的执行次数为T(n),
$T(n) = 2^0+2^1+2^2+…+2^{\lfloor \log_2n \rfloor}=O(2^{\lfloor \log_2n \rfloor})=O(n)$

有些算法在规模相同的情况下，其关键操作的语句频度会因输入的数据值或输入的数据顺序不同而不同，从而估算出的时间复杂度也会不同，为此有最好、最坏和平均时间复杂度之分。

如下算法实现的功能是在数组a[0:n-1]中查找值为x的数据元素。若找到，则返回x在a中的位置；否则，返回-1。求其时间复杂度。

关键操作是x!=a[i]这个比较操作，如果值为x的元素在数组中的第一个位置，则只需要比较一次，如果在第二个位子，就需要比较两次，所以最好时间复杂度是O(1),最坏时间复杂度是O(n)。
平均时间复杂度：$T(n)=\frac 1n\sum_{i=0}^{n-1}(i+1)={n+1\over 2}=O(n)$

通常情况下，讨论算法的好坏是讨论最坏时间复杂度

用冒泡排序法对数组a中的n个整形数据元素进行排序。

最好时间复杂度：T(n)=O(n)
最坏时间复杂度：T(n)=O(n^2)

多项式时间算法的时间复杂度：
$O(1)<O(logn)<O(n)<O(nlogn)<O(n^2)<O(n^3)$
指数时间算法的时间复杂度：
$O(2^n)<O(n!)<O(n^n)$

算法比较举例

最大子序列求和

给定一整数序列A1，A2,…,An（可能有负数），求A1到An的一个子序列Ai到Aj，使得Ai到Aj的和最大。例如：整数序列 -2，11，-4，13，-5，2，-5，-3，12，-9的最大子序列的和为21（从A2到A9）。

  1. 穷举法

    该算法的关键操作是sum += sequence[k];，这行语句的重复执行的次数为：$T(n)=\sum_{i=1}^ni×(n-i+1)=O(n^3)$

  2. 穷举法改进
    $$sum(i,j)=\sum_{k=i}^jA_k$$
    $$sum(i,j+1)=\sum_{k=i}^jA_k+A_{j+1}$$

    $T(n)=\sum_{i=1}^n(n-i)={n(n+1)\over 2}=O(n^2)$

  3. 分治法
    把问题划分成大致相等的两个子问题，然后递归的去求解，这是分的部分，治的部分就是再将这个两个子问题的解合并，并做一些调整后就能得到问题的最终解了。这样求出的最大和的子序列可能在三种位置出现：一种就是整个出现在输入数据的左半部分，另一种是出现在整个数据的右半部分，第三种是跨入输入数据的中部，位于左右两个部分之间。前面两种情况我们可以通过递归求得其解，第三种情况可以通过求出左半部分并且包括左半部分最后一个元素的子序列的最大和，再求出右半部分并且包含右半部分第一个元素的子序列的最大和，然后将两个和相加就可以得到最后的解。

这里用到了递归的方法，时间复杂度的计算比较复杂。详情见书。
最终答案是：$T(n)=n\log_2n+n=O(n\log_2n)$

回文数

如果一个整数，从左往右读和从右往左读都是一样的整数，那么这个整数就是回文数。

  1. 转换字符比较法
    先将整数转为字符串，然后将字符串分割为数组，只需要循环数组的一半长度进行判断对应元素是否相等即可。
  1. 数学解法
    取出每个数字，构造逆序整数，进行比较。
  1. 数字折半逆序比较
    将数字对折，取出后半段进行翻转，看是否与前半段一一对应。
    步骤1：每次对原数x进行取余操作（%10），取出最低位的数字：y=x%10
    步骤2：将最低的数字加到取出数的末尾：revertNum=revertNum*10+y
    步骤3：每取一个最低位数字，x都要自除以10。
    步骤4：判断x是不是小于revertNum，当它小于的时候，说明数字已经对半或者过半了。
    步骤5：判断两个数字位数是否一样。如果不一样，最中间的数字就在revertNum的最低位上，将它除以10以后应该和x相等；如果一样，revertNum和x相等。
